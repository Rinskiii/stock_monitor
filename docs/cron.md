# Cron

## Назначение

`cron` — системный планировщик задач Linux.

После загрузки операционной системы служба `cron` запускается автоматически и
работает в фоновом режиме.

Каждую минуту `cron` проверяет расписание (`crontab`) и, если наступило время,
запускает указанную команду.

В проекте **Stock Monitor** `cron` используется для автоматического запуска
монитора рынка каждые 5 минут.

---

# Проверка работы службы

Проверить, что служба `cron` запущена:

```bash
systemctl status cron
```

Ожидаемый результат:

```text
Active: active (running)
```

Если служба работает, дополнительных действий не требуется.

# Запуск Stock Monitor каждые 5 минут

Пример записи:

```cron
*/5 * * * * cd /home/rinskiii/Desktop/programming/python/my_works/trading/stocks/stock_monitor && /home/rinskiii/miniconda3/envs/stock_monitor_env/bin/python -m monitor.main >> data/logs/cron.log 2>&1
```

---

# Расшифровка расписания

Первая часть записи отвечает только за время запуска.

```text
*/5 * * * *

│   │ │ │ │
│   │ │ │ └── День недели
│   │ │ └──── Месяц
│   │ └────── День месяца
│   └──────── Час
└──────────── Минута
```

Для записи

```text
*/5 * * * *
```

означает:

- `*/5` — каждые 5 минут;
- первая `*` — любой час;
- вторая `*` — любой день месяца;
- третья `*` — любой месяц;
- четвертая `*` — любой день недели.

То есть монитор запускается:

```text
00:00
00:05
00:10
...

19:50
19:55
20:00
20:05
...

23:50
23:55
```

## cd

```bash
cd /home/.../stock_monitor
```

Переходит в каталог проекта.

Это необходимо для корректной работы:

- относительных путей;
- импортов Python;
- записи логов.

---

## Python

```bash
/home/.../stock_monitor_env/bin/python
```

Это путь к исполняемому файлу Python внутри окружения Conda.

Важно понимать, что `cron` **не выполняет**

```bash
conda activate stock_monitor_env
```

Вместо этого сразу запускается Python нужного окружения.

Поэтому монитор работает даже при закрытом терминале.

## Запуск программы

```bash
python -m monitor.main
```

Запускает главный модуль проекта.

Далее выполняется последовательность:

```text
main()

↓

load_watchlist()

↓

↓

↓

download_from_yahoo()
или

↓

проверка консолидации

↓

↓

```

После завершения Python полностью закрывается.

---

## cron.log

```bash
>> data/logs/cron.log
```

Добавляет весь вывод программы в файл

```text
data/logs/cron.log
```

Файл не перезаписывается.

Каждый новый запуск дописывает информацию в конец файла.

---

## Перенаправление ошибок

```bash
2>&1
```

Перенаправляет поток ошибок (`stderr`) в тот же файл, куда записывается обычный вывод (`stdout`).

Это позволяет хранить всю информацию в одном журнале.

# Важные особенности

## Изменение watchlist

Если изменить файл

```text
watchlists/wl01_today.py
```

следующий запуск автоматически использует новый список тикеров.

Перезапуск `cron` не требуется.

---

## Изменение config.py

При изменении:

- `WATCHLIST_NAME`;
- `DATA_SOURCE`;
- `WINDOW_5M`;
- `CONSOLIDATION_QUANTILE`;

новые значения будут использованы при следующем запуске программы.

---

## Если компьютер выключен

Если компьютер выключен в момент запланированного запуска,
например в `20:00`, задача не выполняется позже автоматически.

Следующий запуск произойдет согласно расписанию.

Например:

```text
20:00  компьютер выключен

20:07  компьютер включен

20:10  следующий запуск monitor.main
```

# Почему используется cron

По сравнению с бесконечным циклом:

```python
while True:
    ...
    time.sleep(300)
```

`cron` имеет несколько преимуществ:

- простая настройка;
- не требует постоянно работающего процесса Python;
- автоматически запускает новую копию программы;
- если один запуск завершился с ошибкой, следующий всё равно будет выполнен;
- широко используется на Linux-серверах и рабочих станциях.

Для проекта **Stock Monitor** такой подход является простым, надежным и легко поддерживаемым.
